# 双语学术写作助手

一个面向论文写作的本地优先网页工具。它可以导入中文或英文主稿，自动拆分段落，左右分屏显示中文原文和英文稿，并支持 API 翻译、快速翻译、双语润色、参考论文风格学习、批量翻译和范围导出。

## 主要功能

- 导入主稿：支持 `.docx`、`.txt`、`.tex` 文件。
- 自动识别语言：中文段落进入左侧，英文段落进入右侧，混合内容会保留在中文侧方便人工处理。
- 长段分离写作：把一整段长文本粘贴到任意一侧时，会按论文句子边界自动拆成一句一行；也可以选中段落后点击“拆分当前段落”。
- TeX 清理：导入 `.tex` 时会尽量去除命令、环境、表格控制符和模板残留，只保留正文内容。
- 双向翻译：中文译英文、英文译中文都支持。
- 自动翻译方式：
  - API 自动翻译：输入变化 4 秒后调用配置的模型接口。
  - 快速自动翻译：使用免密快速翻译来源，适合临时对照。
  - 关闭自动翻译：只保留手动编辑。
- 批量翻译：勾选多个段落后，点击“翻译当前段落”会合并为一次编号请求，并按从上到下回填，减少 API 调用次数。
- 双语润色：支持“润色当前中文”和“润色当前英文”。
- 修订说明：AI 返回的修订说明会显示在对应中文或英文格子的下方，不会混入正文。
- 参考论文库：最多导入 10 篇参考论文，用于提取常用术语和句式风格。
- 导出范围：可输入起始段和结束段，只导出指定范围。
- 导出格式：
  - 中文 TXT
  - 英文 TXT
  - 双语对照 TXT
  - 中文 DOCX
  - 英文 DOCX

## 本地运行

```bash
npm install
npm run dev
```

默认访问：

```text
http://127.0.0.1:5173/
```

## 构建与测试

```bash
npm test -- --run
npm run build
```

构建产物会输出到 `dist/`。

## API 配置说明

右侧“API 设置”中可以选择常见厂商预设，也可以使用自定义 OpenAI 兼容接口。

需要注意：

- 当前版本是纯前端网页，API 密钥只保存在当前浏览器会话中，不会写入仓库。
- 如果部署成公开网页，前端直连模型 API 可能暴露用户自己输入的 key，也可能遇到 CORS 限制。
- 更适合公开长期使用的方案是后续增加后端代理服务：前端只调用自己的后端，由后端保存密钥、限流、记录用量并转发到模型厂商。

## GitHub Pages 部署

当前仓库使用 `gh-pages` 静态分支发布公网网页，不依赖 GitHub Actions workflow 权限。

部署流程：

```bash
npm test -- --run
npm run build
git subtree push --prefix dist origin gh-pages
```

首次部署后，在仓库的 `Settings -> Pages` 中选择：

- Source: Deploy from a branch
- Branch: `gh-pages`
- Folder: `/`

如果 GitHub 账号 token 具备 `workflow` 权限，也可以后续改为 GitHub Actions 自动部署。

## 使用建议

- 长论文建议先导入全文，再按段落勾选翻译。
- 如果复制的是整段连续文字，可直接粘贴到中文或英文编辑格，系统会先拆句，再用于后续逐句翻译或润色。
- 如果担心 API 429 或速度慢，优先使用“勾选多段 + 翻译当前段落”，它会把选中段落合并为一次请求。
- 对正式投稿文本，建议先快速翻译，再用 API 翻译或润色精修。
- 参考论文最好选择同领域、同类型论文，例如综述配综述、方法论文配方法论文。
